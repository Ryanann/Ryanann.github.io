Vim开发环境搭建

2020/7/4 posted in  vim

Vim的使用在日常开发中十分重要，相关的插件也十分丰富。本文就面向MacOS和Linux系统，对其中几个较知名插件的安装进行介绍。提到Vim的插件，当然少不了有最难安装Vim插件之称的YouCompleteMe。这个插件的安装在不同的系统中会遇到不同的问题，文章会介绍本人两次安装的过程。

建议大家在安装插件时更新Vim版本，至少要确保版本高于7.4，否则在安装过程中会出现各种问题，尤其是YouCompleteMe插件。

Vundle

Vundle是一个Vim插件管理器，可以帮助我们方便地进行插件的安装、更新和删除。

运行如下指令安装Vundle：

git clone https://github.com/gmarik/Vundle.vim.git ~/.vim/bundle/Vundle.vim

项目clone完毕后在~/.vimrc中添加配置：

set nocompatible              " required
filetype off                  " required
dd
set rtp+=~/.vim/bundle/Vundle.vim
call vundle#begin()
Plugin 'gmarik/Vundle.vim'
call vundle#end()            " required
filetype plugin indent on    " required

最后在vim中运行:PluginInstall就算安装完成了。之后再安装插件只需要在call vundle#begin()和call vundle#end()中添加语句即可。

YouCompleteMe

本人在安装该插件时，在MacOS系统和Linux系统中遇到了不同的情况。

MacOS

如果需要支持go项目自动补全等，需要在~/.bash_profile文件中正确配置GOROOT、GOPATH、GOBIN路径，且路径中不能包含文件链接

在~/.vimrc中添加如下语句：

Plugin 'Valloric/YouCompleteMe'

执行PluginInstall后提示YouCompleteMe unavailable: unable to load Python。

于是重新安装Vim，并设定当前用户使用新安装的Vim：

brew install vim

命令执行后，可以看到新vim的路径，如/usr/local/bin/vim
然后在~/.bash_profile中添加配置，指定当前用户运行的vim：

alias vim='/usr/local/bin/vim'

完成vim的重新安装后，首先删除YCM插件（删去vimrc中的相关行，然后执行:PluginClean）

然后重新安装YCM插件，安装完成后发现之前的错误提示消失了，取而代之的是The ycmd server SHUT DOWN (restart with ':YcmRestartServer'). YCM core library not detected; you need to compile YCM before using it. Follow the instructions in the documentation

这就是YCM安装的复杂之处了，我们需要对它进行编译安装，具体指令如下：

# 安装cmake
brew install cmake

# 准备编译文件夹
cd ~
mkdir ycm_build
cd ycm_build

# 生成makefiles文件（若不需要支持C/C++的自动补全）
cmake -G "Unix Makefiles" ~/.vim/bundle/YouCompleteMe/third_party/ycmd/cpp

# 生成makefiles文件（若需要支持C/C++的自动补全）
cmake -G "Unix Makefiles" -DUSE_SYSTEM_LIBCLANG=ON . ~/.vim/bundle/YouCompleteMe/third_party/ycmd/cpp

# 编译
cmake --build . --target ycm_core --config Release

这样就算安装成功了，此时YCM可以支持对Python项目（若选择了第二种生成makefiles文件还可以支持c/c++项目）的自动补全和代码跳转，若需要进一步对go等项目进行支持，还需进一步配置：

cd ~/.vim/bundle/YouCompleteMe
python3 install.py --go-completer
# 更多安装可以执行python3 install.py --help查看

MacOS安装YCM的主要问题就在于系统默认Vim安装后会找不到Python支持。

Linux

如果需要支持go项目自动补全等，需要在~/.bash_profile文件中正确配置GOROOT、GOPATH、GOBIN路径，且路径中不能包含文件链接。本人刚开始采用~/go配置相关路径，发现插件不能正确使用，后改为从根目录开始的路径，才正确装上。

在~/.vimrc中添加如下语句：

Plugin 'Valloric/YouCompleteMe'

执行PluginInstall后提示The ycmd server SHUT DOWN (restart with ':YcmRestartServer'). YCM core library not detected; you need to compile YCM before using it. Follow the instructions in the documentation

sudo apt-get install cmake  
sudo apt-get install python-dev python3-dev

因为本人使用的Linux系统中没有Clang，所以需要下载相关包以让YCM支持c/c++项目的自动补全：

wget http://releases.llvm.org/9.0.0/clang+llvm-9.0.0-x86_64-linux-gnu-ubuntu-14.04.tar.xz

xz -d clang+llvm-9.0.0-x86_64-linux-gnu-ubuntu-14.04.tar.xz
tar -xvf clang+llvm-9.0.0-x86_64-linux-gnu-ubuntu-14.04.tar

编译：

cd ~
mkdir ycm_build
#将下载的clang移到一个自己建的llvm目录中
mkdir -p ycm_temp/llvm_root_dir
mv ~/clang+llvm-9.0.0-x86_64-linux-gnu-ubuntu-14.04/* ~/ycm_temp/llvm_root_dir/
cd  ycm_build
cmake -G "Unix Makefiles" -DUSE_SYSTEM_BOOST=ON -DPATH_TO_LLVM_ROOT=~/ycm_temp/llvm_root_dir ~/.vim/bundle/YouCompleteMe/third_party/ycmd/cpp

当然，如果不需要支持c/c++项目的代码补全和跳转，则不需要下载llvm、clang，直接使用如下指令编译即可：

cd ~
mkdir ycm_build
cd ycm_build
cmake -G "Unix Makefiles" ~/.vim/bundle/YouCompleteMe/third_party/ycmd/cpp
cmake --build . --target ycm_core --config Release

同样地，如果需要支持go项目：

cd ~/.vim/bundle/YouCompleteMe
python3 install.py --go-completer
# 更多安装可以执行python3 install.py --help查看

NERDTree

NERDTree插件可以展示项目的目录结构，其安装相对简单。

在~/.vimrc文件的call vundle#begin()和call vundle#end()中添加语句Plugin 'scrooloose/nerdtree'，然后执行:PluginInstall

这样就算安装成功了，比起YCM确实简单了不少。可以继续在~/.vimrc中添加NERDTree相关配置：

" NERDTree config
" open a NERDTree automatically when vim starts up
autocmd vimenter * NERDTree
"open a NERDTree automatically when vim starts up if no files were specified
autocmd StdinReadPre * let s:std_in=1
autocmd VimEnter * if argc() == 0 && !exists("s:std_in") | NERDTree | endif
"open NERDTree automatically when vim starts up on opening a directory
autocmd StdinReadPre * let s:std_in=1
autocmd VimEnter * if argc() == 1 && isdirectory(argv()[0]) && !exists("s:std_in") | exe 'NERDTree' argv()[0] | wincmd p | ene | endif
"map F2 to open NERDTree
map <F2> :NERDTreeToggle<CR>
"close vim if the only window left open is a NERDTree
autocmd bufenter * if (winnr("$") == 1 && exists("b:NERDTree") && b:NERDTree.isTabTree()) | q | endif

配置完成后，使用vim打开文件时，会自动打开NERDTree。
